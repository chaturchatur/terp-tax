# Plan: CampusTax — AI Tax Filer for Students

## Context

Building for **Claude Hackathon Track 2 (Campus Intelligence & Equity)**, scoped specifically to **UMD (University of Maryland, College Park) students**. Filing taxes is one of the sharpest equity gaps on campus: first-gen students have no family guide, and international students on F-1/J-1 visas (who _must_ file Form 8843 even with $0 income) typically pay Sprintax $50+ or skip filing entirely. The goal is a tool where a UMD student drops in their W-2 / 1098-T / 1042-S, answers a few guided questions, and walks out with filled, downloadable **federal + Maryland state** PDFs — with every field explained in plain English sourced from the official IRS and Comptroller of Maryland instructions.

**Target filer population:**

- US residents: Federal Form 1040 + **Maryland Form 502** (resident) with local/county tax (Prince George's County rate for College Park addresses, selectable for commuters).
- Non-resident international students (F-1/J-1): Federal Form 1040-NR + Form 8843 + **Maryland Form 505** (non-resident) where MD-source income exists.

**Out of scope:** E-filing (requires IRS EFIN registration), other states (MD only), non-US tax systems, payroll-portal integrations, part-year resident MD returns (Form 502 + 505 combo — cut for weekend scope).

**The NotebookLM question:** No public API. We replicate the "ask the instructions" experience natively by feeding the official IRS instruction PDFs to Claude as context when a user hits "Explain this field" — better UX than shipping out to an external notebook. A public NotebookLM notebook loaded with IRS pubs can be linked as a secondary "study" resource on a help page (5-min add-on, optional).

---

## Stack

- **Next.js 15 (App Router) + TypeScript + Tailwind + shadcn/ui** — single app, API routes for Claude calls
- **Anthropic SDK** (`@anthropic-ai/sdk`) with **Claude Sonnet 4.6** as default, **Opus 4.7** for the residency/eligibility reasoning step
- **Claude vision + PDF input** for form extraction (native PDF support — no OCR library needed)
- **pdf-lib** for filling the official IRS fillable PDFs (AcroForm fields)
- **Zod** for schema validation of extracted form data
- **Vercel AI SDK** (`ai`) only if we want streaming UI for the chat/explain panel — otherwise raw Anthropic SDK

---

## User flow

1. **Landing** — "Are you a US citizen/resident or an international student?" (branches to 1040 vs 1040-NR track).
2. **Residency wizard** — 4–5 questions implementing the IRS Substantial Presence Test for ambiguous cases (visa type, years in US, days present). Claude reasons over the answers and explains the determination.
3. **Document intake** — drag-and-drop zone accepting W-2, 1098-T, 1099-INT, 1042-S (PDF or image). Claude vision extracts fields into a typed schema. Extracted values flow directly into the form view (step 4) — **no separate confirmation table**.
4. **Interactive form view (the core UX)** — the app renders each required return as a faithful on-screen form (1040, 1040-NR, 8843, MD 502, MD 505) using an HTML layout that visually echoes the real IRS/MD form (numbered lines, section headers, two-column layout). Every field is a live input, and every line has an **ⓘ** icon next to it:
   - **Prefilled fields** (from uploaded docs or computed from other lines) show the value with its source badge ("from W-2 Box 1" / "computed: line 9 – line 12"). User can override.
   - **User-entry fields** are editable inputs.
   - **Computed fields** update live as inputs change (deterministic `lib/tax/` functions run client-side; Claude never touches the math).
   - **Clicking ⓘ** slides in a right-side panel with the plain-English explanation for that exact line, generated by Claude from the official IRS or MD Comptroller instructions PDF, with a page-number citation and a "Show original instruction text" expander. Cached per (formId, lineId).
   - Tabs at the top switch between **Federal** and **Maryland** returns; a progress strip shows which lines are still blank/unreviewed.
   - Guided questions (dependents, scholarship use, treaty country, MD county) are asked **inline** as the relevant lines are focused — not in a separate pre-screen — so context stays with the form.
5. **Download** — when all required lines are green-checked, one button produces a zip of filled official federal + Maryland PDFs (generated server-side via pdf-lib from the on-screen values). Plain-English summary ("Federal refund $847, MD refund $312 — here's why") and next-steps checklist (sign, mail federal to Kansas City, mail MD to Annapolis, keep copies).

## Implementation notes

### Extraction (`api/extract`)

Use Claude's native PDF/image input. Pass a Zod-derived JSON schema in the prompt and use **tool use** with a single `return_extracted_fields` tool whose `input_schema` matches the target form schema — guarantees structured output, no regex parsing.

### Explain (`api/explain`)

For each IRS and MD form we pre-download the instructions PDF into `public/forms/`. The explain endpoint sends `{instructions_pdf, line_number, line_label, user's_current_value, filer_context}` to Claude with a prompt like "Using only the attached instructions, explain line {N} in plain English to a UMD student. Cite page numbers." Claude's PDF support handles it natively — no chunking/embedding needed for a hackathon. Cache responses by `(formId, lineId)` in `unstable_cache` since they're largely deterministic per line. Client calls this lazily — only when the user actually clicks ⓘ — so we don't pay for explanations nobody reads.

### Interactive form rendering (`components/FormRenderer.tsx`)

Each form is defined as a **declarative layout** (`components/forms-layout/layout-1040.ts` etc.) — an array of sections and lines with `{id, lineNumber, label, kind: 'input'|'prefill'|'computed', compute?: (state) => value, source?: 'w2.box1'|...}`. `FormRenderer` walks this and renders `FormLine` components in an IRS-style two-column grid (line number | label + input | ⓘ). Computed lines re-evaluate reactively when dependencies change via a small Zustand store keyed by `{formId, lineId}`. This keeps the "real IRS form look" without us hand-coding JSX per form.

### Compute (`api/compute`)

Tax math must be **deterministic** — do NOT have Claude compute refund amounts. Pure TypeScript functions in `lib/tax/` apply 2025 federal tax tables, standard deduction, credits — and then feed federal AGI into the Maryland computation:

- **MD starts from federal AGI**, applies MD-specific additions/subtractions (e.g. subtract 529 plan contributions, subtract some scholarship amounts), applies MD standard deduction (15% of MD AGI, bounded $1,700–$2,550 single for 2024 — verify current year at implementation time), applies MD tax brackets, then adds **county/local tax** on top using the filer's Maryland county of residence (Prince George's ~3.2%, Montgomery ~3.2%, etc. — table in `md-local-rates.ts`).
- Non-residents file Form 505 only if they had MD-source income (most F-1s with a UMD campus job do).

Claude's role is limited to: (a) residency determination (federal SPT + MD domicile question), (b) which credits the user qualifies for (AOTC vs LLC, MD poverty-level credit via 502CR, treaty benefits), (c) plain-English explanations. Every Claude decision surfaces a rationale in the UI.

### PDF filling (`lib/pdf/`)

Both the IRS and the Maryland Comptroller publish fillable PDFs with named AcroForm fields. Map computed return → field names once per form (one-time dev effort: open PDF, inspect field names with pdf-lib). `pdf-lib` writes values and returns bytes; download endpoint zips federal + MD together.

### Residency (Substantial Presence Test)

Deterministic in `lib/tax/spt.ts` (it's a formula: days this year + 1/3 last year + 1/6 year before ≥ 183). F-1/J-1 students get 5 exempt years — Claude handles the edge cases and explains the result.

## Demo story (2-min pitch)

1. Open app → "I'm a UMD student on an F-1 visa" → answer 4 visa questions → app determines "non-resident alien, must file 8843 + 1040-NR + MD Form 505."
2. Drop in W-2 (UMD payroll) and 1042-S PDFs → fields extract and the next screen is **the actual 1040-NR rendered as an interactive form**, already prefilled.
3. Scroll the form — click ⓘ on "Line 1a: Wages" → right panel slides in with Claude's plain-English explanation + IRS Pub citation. Change a number — totals recompute live.
4. Switch to **Maryland** tab → MD 505 shown in the same style, prefilled from federal AGI; click ⓘ on line 17 → Claude cites the MD Comptroller booklet.
5. Download zip: filled official 1040-NR + 8843 + MD 505 PDFs that match what was on-screen.
6. Tagline: _"Sprintax charges $51 for federal and another $40 for Maryland. We're free for UMD students, explain every line, and built in 48 hours with Claude."_

## Verification

- **Extraction**: use a sample W-2 (IRS publishes specimen forms in Pub 15-A) → confirm all 20+ boxes extract correctly.
- **Computation**: cross-check refund for 3 fixture scenarios (UMD student W-2 only → 1040 + MD 502; student with 1098-T + AOTC → 1040 + MD 502; F-1 with UMD campus-job W-2 + 1042-S + treaty benefit → 1040-NR + 8843 + MD 505) against hand-computed expected values for **both federal and Maryland** including local county tax.
- **PDF fill**: open each downloaded PDF (federal + MD) in Preview / Acrobat — all fields populated, layout intact, signature line blank.
- **Explain**: click 10 random fields, verify Claude cites real IRS instruction page numbers (spot-check against the PDF).
- **End-to-end**: run `pnpm dev`, complete the full NR flow in the browser, download PDF, visually inspect.

## Cut list (if running out of time, in order)

1. Skip 1099-INT extraction (manual entry only).
2. Skip Schedule A / MD itemized deductions — standard deduction only (fine for 95% of students).
3. Skip treaty benefit auto-detection for NR — user picks country, we apply.
4. Skip MD Form 502CR credits beyond the basic refundable EIC — table-lookup only.
5. Skip the "next steps / where to mail" screen — a static page is fine.
6. Last resort: ship **federal only** and keep MD as a post-hackathon extension — but this weakens the UMD-specific story, so protect it.
